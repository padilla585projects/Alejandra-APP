================================================================================
  QUÉ ES EL AGENTE ALEJANDRA
================================================================================

Alejandra es un agente IA (Claude Sonnet via Anthropic API) integrado en la
app Alejandra APP. Tiene dos partes:

1. LA APP PRINCIPAL (worker.js)
   - URL: https://alejandra-app-api.alejandra-app.workers.dev
   - Alejandra vive aquí desde las versiones antiguas (devAIChat, handleDevAI)
   - Chat accesible desde la app móvil (index.html) — botón 🤖

2. EL AGENTE INDEPENDIENTE (alejandra-agente/worker.js)
   - URL: https://alejandra-agente.alejandra-app.workers.dev
   - Worker Cloudflare separado con lógica NEXUS avanzada
   - Panel de control: https://padilla585projects.github.io/Alejandra-APP/alejandra-panel.html
   - Token acceso: ver sección "AUTENTICACIÓN" más abajo

IMPORTANTE: Ambos comparten el MISMO D1 (alejandra-db). Son la misma Alejandra.

================================================================================
  ARQUITECTURA DEL AGENTE (alejandra-agente/worker.js)
================================================================================

NEXUS ROUTER:
  1. Haiku clasifica el mensaje → determina experto (simple/app/tecnico/web/reflexion/completo)
  2. Se carga el prompt dinámico (NEXUS_MODULES) para ese experto
  3. Se llama a Sonnet con las herramientas del experto
  4. Bucle de hasta MAX_ITERATIONS si hay tool calls

NEXUS_MODULES (fragmentos de prompt):
  base       — identidad Alejandra, reglas generales
  app        — contexto de la app industrial (bobinas, equipos, etc.)
  tecnica    — stack técnico, tablas D1, worker.js
  nexus      — cómo funciona el router NEXUS
  evolucion  — autonomía, self-audit
  web        — búsqueda web con OpenAI
  reflexion  — auto-mejora y memoria
  decision   — autoconciencia y toma de decisiones autónoma
  formato    — cómo formatear respuestas

NEXUS_EXPERTS (perfiles):
  simple    → Haiku, max 512 tokens, módulos [base, formato]
  app       → Sonnet, max 1024, módulos [base, app, formato]
  tecnico   → Sonnet, max 2048, módulos [base, app, tecnica, nexus, evolucion, formato]
  web       → Sonnet, max 1536, módulos [base, web, formato]
  reflexion → Sonnet, max 2048, módulos [base, app, tecnica, nexus, evolucion, reflexion, decision, formato]
  completo  → Sonnet, max 3072, todos los módulos

HERRAMIENTAS DISPONIBLES (por experto):
  simple:    []
  app:       [memory_read]
  tecnico:   [leer_estado, memory_read]
  web:       [buscar_web]
  reflexion: [memory_save, memory_read, propose_mejora, buscar_web, tomar_decision, leer_estado]
  completo:  [buscar_web, memory_read, leer_estado]

TODAS LAS HERRAMIENTAS:
  memory_save       — guarda en alejandra_memoria
  memory_read       — lee alejandra_memoria
  propose_mejora    — propone cambios de código (para revisión humana)
  buscar_web        — búsqueda con OpenAI gpt-4o-mini + web_search_preview
  leer_estado       — lee config, conteo memorias, últimas decisiones, logs
  tomar_decision    — registra decisión; si tipo=config + confianza≥0.8 → aplica automáticamente

================================================================================
  ENDPOINTS DEL AGENTE
================================================================================

PÚBLICOS (sin auth):
  GET  /health                  → {status, version, nexus, reflexion, decisiones, web_search}
  POST /api/chat                → {mensaje, usuario_id, [empresa_id, canal]} → {respuesta, experto, iteraciones}

ADMIN (requieren Authorization: Bearer <token>):
  GET  /api/admin/config        → {modo, auto_fix, max_iterations}
  POST /api/admin/config        → {modo, auto_fix, max_iterations} → {ok}
  GET  /api/admin/logs          → [{id, accion, parametros, resultado, status, created_at}]
  GET  /api/admin/memoria       → [{id, tipo, titulo, contenido, importancia, created_at}]
                                  ?tipo=aprendizaje|decision|etc  ?limit=50
  GET  /api/admin/chat          → [{canal, rol, contenido, created_at}]
                                  ?canal=web|telegram|panel  ?limit=100
  GET  /api/admin/gastos        → {periodo_dias, total_usd, total_eur, por_modelo, por_dia}
                                  ?dias=7|30|90
  GET  /api/admin/tokens        → [{id, descripcion, tipo, activo, created_at}]
  POST /api/admin/tokens        → {nombre, token_valor} → {ok}  (crear usuario)
  DELETE /api/admin/tokens      → {id} → {ok}  (revocar acceso)
  POST /api/admin/tokens/change → {token_nuevo} → {ok}  (cambiar propia contraseña)
  POST /api/reflexion           → {token} → {ok}  (lanzar reflexión autónoma)

  AUTENTICACIÓN
================================================================================

El panel usa tokens almacenados en alejandra_tokens (D1).
La función verificarAdminToken comprueba:
  1. Si env.ADMIN_TOKEN existe y coincide → autorizado (env var como respaldo)
  2. Si no → consulta alejandra_tokens WHERE token=? AND tipo='admin' AND activo=1

TOKEN ACTUAL: 'alejandra2026' (o el que haya configurado el usuario en el panel)

HISTORIA DEL BUG (para entender por qué hay dos mecanismos):
  - migrate_001 usaba INDEX inline en CREATE TABLE (inválido en SQLite/D1)
  - Eso hacía fallar la migración silenciosamente → alejandra_tokens vacía
  - migrate_003 lo corrige: crea tablas con sintaxis correcta + INSERT OR REPLACE del token
  - env.ADMIN_TOKEN (wrangler secret) se setea en el workflow como respaldo

================================================================================
  REGLAS PARA MODIFICAR EL AGENTE
================================================================================

1. El agente está en alejandra-agente/worker.js (NO en worker.js raíz)
2. Siempre subir versión en health endpoint y comentario de cabecera
3. Si cambias tablas D1 → crear migrate_00X_*.sql y añadirlo al workflow
4. Verificar que version.json / sw.js / index.html están en sync antes de push
5. El agente usa ES modules (export default { fetch() })
6. No usar `wrangler.toml` del raíz para el agente — usar alejandra-agente/wrangler.toml
7. Para desplegar solo el agente manualmente:
     cd alejandra-agente && npx wrangler deploy
8. Para ver logs en tiempo real:
     npx wrangler tail --name alejandra-agente
