================================================================================
  DECISIONES TÉCNICAS IMPORTANTES
================================================================================

1. INSERT...RETURNING no funciona bien en D1.
   Siempre usar .run() + meta.last_row_id para obtener el ID insertado.

2. El binding D1 se pierde en deploy via curl si no se incluye en metadata.
   Con `npx wrangler deploy` es automático (lee wrangler.toml). USAR WRANGLER.

3. apiCall(path, options) — SIEMPRE 2 argumentos.
   MAL: apiCall('/ruta', 'POST', body)
   BIEN: apiCall('/ruta', { method: 'POST', body: JSON.stringify(data) })

4. Credenciales Git: usar `git push origin main` en app nueva (OAuth guardado).
   App vieja: usar -c credential.helper= con token en URL.

5. sendBeacon debe usar Blob({type:'application/json'}) — string plano no funciona.

6. version.json debe actualizarse SIEMPRE junto con sw.js CACHE y badge index.html.

7. fichajes.estado (no 'tipo') — valores: presencia / presente / ausencia /
   retraso / baja / vacaciones / festivo. Default DB: 'presente'.

8. getFichajes devuelve array plano (no {fichajes:[]}). Manejar con:
   Array.isArray(data) ? data : (data.fichajes || [])

9. getTrabajadores devuelve array plano (merge usuarios app + personal_externo).
   Cada item tiene campo 'tipo': 'app' | 'externo'.

10. registrarHistorialHerr / registrarHistorialKitHerr — nombres únicos para evitar
    conflicto con registrarHistorial (bobinas).

11. empresa_admin: tiene acceso total a datos de su empresa.
    getAuth() devuelve isEmpresaAdmin: sesion.rol === 'empresa_admin'.

12. AHORA() helper en worker: new Date().toISOString().slice(0,19).replace('T',' ')
    Usar para todos los timestamps de fechas automáticas.
