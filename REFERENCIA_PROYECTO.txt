================================================================================
  DECISIONES TÉCNICAS IMPORTANTES
================================================================================

1. INSERT...RETURNING no funciona bien en algunas versiones de D1.
   Siempre usar .run() + meta.last_row_id para obtener el ID insertado.

2. El binding D1 se pierde en cada deploy via REST API si no se incluye en metadata.
   SIEMPRE incluir el binding en el curl de deploy (ver comandos arriba).

3. apiCall(path, options) — el segundo argumento es el objeto fetch options completo.
   MAL: apiCall('/ruta', 'POST', body)
   BIEN: apiCall('/ruta', { method: 'POST', body: JSON.stringify(data) })

4. El credential helper de Windows puede usar el token de padilla585projects
   para el repo de padilla585. Siempre usar -c credential.helper= en git push
   y poner el token directamente en la URL.

5. sendBeacon debe usar Blob({type:'application/json'}) — string plano no funciona.

6. version.json debe actualizarse siempre que se cambie la versión en sw.js/index.html.

7. Sugerencias: la tabla tiene columna 'estado' (pendiente/en_progreso/resuelto).
   El worker soporta filtros ?estado= y ?categoria= en GET /sugerencias.

8. empresa_admin en nueva app: tiene acceso total a sus datos de empresa pero
   NO tiene acceso a config global (Sheets, Logs sistema, etc.).
   getAuth() devuelve isEmpresaAdmin: sesion.rol === 'empresa_admin'.
